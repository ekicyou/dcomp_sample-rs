# Requirements Document

## Project Description (Input)
event-hit-test の孫仕様：αマスクによるピクセル単位ヒットテスト

画像がヒットテスト対象となった場合に、αが50%以上（α ≧ 128）の領域に対してヒットさせるドット単位判定を実現する。画像読み込み完了時に対応する2値マスク画像（ヒットマスク）を生成し、ヒットテスト時にピクセル単位での判定を行う。

### 背景
デスクトップマスコットアプリケーションでは、キャラクター画像の透明部分をクリックしてもヒットしないことが期待される。単純な矩形判定では透明部分もヒット対象となってしまうため、αチャンネルを考慮したピクセル単位の判定が必要。

### 対応する親仕様（event-hit-test）の要件
- **透明領域のヒット対象外オプション**: αが閾値未満の領域をヒット対象外とする

### 主な機能
- 画像読み込み時の2値マスク自動生成（α ≧ 50%でヒット）
- ピクセル座標からのヒット判定
- マスク画像のキャッシュ管理
- 閾値のカスタマイズ（デフォルト: 50%）

### 技術的考慮事項
- マスク画像は1ビット/ピクセルで効率的に保持
- 画像サイズに応じたメモリ使用量の最適化
- スケーリング時のマスク再計算
